关于ES6中，let const 个人理解

ES6中新增了两个声明变量的方式，let和const。 下面将对let、const以及var做粗略比较。 （处于学习阶段，如果错误，请不吝指正）

let、var： let我理解为块级作用域，不存在声明提升。 在一个script标签内部使用let声明一个变量，这个变量只存在于这个script的作用域中
let name = 'Geek_Chen'; 此时的name不是window上的一个变量，是存在于foo函数中的一个变量。
var name = 'Geek_Chen'; 此时的name是window上的一个变量

const: 用于声明一个不改变内存地址的常量。 曾在面试中遇到这样一个问题： const obj = {}; 然后对obj进行修改，譬如：
obj.name = 'Geek_Chen' 、 会发生什么
处于初级学习阶段的我被难住了，因为那时对于js中的堆栈问题始终不是很明白。

answer：可以对obj进行正常的添加修改等操作。
请注意，之前我们对const的解释是 用于声明一个不改变内存地址的常量。 也就是说，我们声明了obj后对其进行修改，修改的只是其在栈中的属性，并没有对其所在堆中的指针（个人理解为 每个引用数据类型都在堆中有自己的内存空间，也就是指针） 做修改，所以 在const了obj之后，是可以对其进行修改的。

var: 使用var声明变量，很容易引起全局变量污染问题，虽然js中有垃圾回收机制，但作为初级学者是不可控的，所以我们在日常开发中，应该尽量减少对var的使用
